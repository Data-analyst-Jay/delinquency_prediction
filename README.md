# Credit Card Delinquency Prediction Project

This repository contains a data science and machine learning project undertaken as part of a Tata micro-internship focused on GenAI-based data analysis. The goal of the project is to build an explainable machine learning model that predicts the probability of a credit card customer becoming 30+ days past due (DPD) in the next billing cycle, enabling proactive intervention and reducing credit losses.

## 📌 Problem Overview
- **Business Context:** Geldium Finance is experiencing a high credit card delinquency rate. Early identification of potential delinquents allows the company to intervene with reminders, calls, or credit limit reviews.
- **Objective:** Develop a model that forecasts 30+ DPD delinquency for the next cycle. Focus is on high recall for the delinquent class while keeping false positives in check.
- **Ideal Model:**
  - Recall (Delinquent) ≥ 75%
  - Precision ≥ 60%
  - ROC-AUC ≥ 0.80
  - Cost reduction ≥ 15% compared to a no-model baseline
  - Acceptable false positive rate: 20–30%

## 🛠 Key Steps Covered
1. **Data Loading & EDA** – Inspecting structure, distributions, missing values, and class balance.  
2. **Cleaning & Feature Engineering** – Imputing missing values, encoding categorical fields, creating flags and derived features (e.g., stress index, trend score, consecutive misses).  
3. **Exploratory Analysis** – Correlation checks, bivariate statistics, and visualization.  
4. **Modeling** – Splitting data, scaling, and training logistic regression with class balancing; experimenting with SVM and grid search for hyperparameter tuning.  
5. **Evaluation** – Using metrics like accuracy, precision, recall, ROC-AUC, confusion matrices and adjusting thresholds to balance recall vs false positive rate.  
6. **Model Persistence** – Saving pipelines with `joblib` for later scoring.
